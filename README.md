Привет! Это программа, которая:

Смотрит видео с камеры (или любой файл)

Находит ценники с помощью нейросети YOLO

Улучшает их качество (убирает шум, повышает резкость, может даже применить Real-ESRGAN для супер-разрешения)

Отправляет картинку в Yandex Vision OCR, чтобы распознать текст

Парсит название, цену и единицу измерения

Сопоставляет с вашей базой товаров (db_hack.csv) через нечёткий поиск + жёсткие семантические фильтры (никаких случайных совпадений!)

Отслеживает каждый ценник на видео (трекинг) и выбирает лучший результат за всё время

В конце выдаёт CSV-файл со всей найденной информацией

🚀 Как этим пользоваться?
1. Установите Python 3.9 или выше
Скачать можно с официального сайта.

2. Скопируйте все файлы проекта в одну папку
detect_prices.py (основной скрипт, назовите как хотите)

db_hack.csv – ваша база товаров (обязательно!)

Папка datasets с файлом best.pt (обученная модель YOLO)

3. Установите зависимости
Откройте терминал (командную строку) в папке проекта и выполните:

bash
pip install pandas opencv-python numpy requests ultralytics rapidfuzz realesrgan basicsr torch torchvision
Если что-то не установится, можно разбить на части:

bash
pip install pandas opencv-python numpy requests
pip install ultralytics
pip install rapidfuzz
pip install realesrgan basicsr
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu   # если нет видеокарты
Примечание: torch может весить много, но он нужен для Real-ESRGAN и YOLO. Если не хотите использовать улучшение картинки, позже в коде можно отключить APPLY_AI_ENHANCE = False.

4. Получите API-ключ Yandex Vision OCR
Программа использует бесплатную OCR от Яндекса. У каждого нового аккаунта есть стартовые гранты (обычно хватает на тысячи распознаваний).

Как получить ключ:

Зарегистрируйтесь в Яндекс.Облаке.

Создайте платёжный аккаунт (привяжите карту – с вас не спишут, пока вы не потратите грант, но это обязательно).

Перейдите в каталог и создайте каталог (например, my-ocr).

В каталоге перейдите в раздел API-ключи → Создать API-ключ.

Скопируйте полученный ключ (выглядит как AQVN...).

Там же посмотрите ID каталога (строка вида b1g1...).

Теперь откройте скрипт и найдите строки:

python
YANDEX_API_KEY   = "AQVN01X7SsqPYYWmaN-wO2fCpKlenaLVQPIv1twc"
YANDEX_FOLDER_ID = "b1g1f5c07ffpj7v067d0"
Замените их на свои.

Важно: Если оставить демо-ключи, OCR работать не будет (они мои и могут быть заблокированы). Обязательно поставьте свои!

5. Запустите программу
bash
python detect_prices.py
Сначала откроется диалог выбора видеофайла (можно выбрать .mp4, .avi, .mov, .mkv).

Если диалог не появился – введите путь к видео вручную в консоли.

Начнётся обработка. Вы увидите окно с детекцией ценников (можно нажать Q для досрочного выхода).

В папках detected_frames и improved_roi будут сохраняться промежуточные результаты.

После завершения видео появится файл final_results.csv с итоговой таблицей.

⚙️ Основные настройки (можно менять в коде)
В разделе # НАСТРОЙКИ ОБРАБОТКИ:

Параметр	Что делает	Рекомендация
TARGET_FPS	Сколько кадров в секунду обрабатывать (меньше – быстрее)	5.0 (хороший баланс)
CONF_THRESHOLD	Порог уверенности YOLO	0.25 (можно поднять до 0.4, если много ложных)
APPLY_AI_ENHANCE	Улучшать картинку через Real-ESRGAN (очень ресурсоёмко)	False, если компьютер слабый
TRACK_TIMEOUT	Через сколько кадров считать, что ценник пропал	30
FUZZY_THRESHOLD	Минимальный score для нечёткого поиска	60
APPLY_DB_MATCH	Искать ли совпадения в БД	True
📁 Структура выходных папок
detected_frames/ – полные кадры с нарисованными bounding boxes (раз в TARGET_FPS секунд).

improved_roi/ – вырезанные и улучшенные картинки ценников.

f{frame}_t{track}.jpg – каждый кадр каждого трека.

f{frame}_t{track}.json – распознанный текст и результат БД для этого кадра.

track_{id}_final.json – итоговый лучший результат для каждого уникального ценника.

final_results.csv – главная таблица: одна строка = один найденный товар.

🧠 Как работает сопоставление с БД?
Сначала ищем штрихкод (8–14 цифр) – если находим, доверяем на 100%.

Если нет – очищаем текст от цен, единиц и мусорных слов.

Выполняем fuzz.WRatio с базой, но результат принимаем только если:

Есть хотя бы одно значащее слово (длина ≥3, не стоп-слово) из запроса в названии кандидата.

word_overlap > 0 (пересечение значимых слов).

Дополнительно проверяем категорию (например, "напиток" не перепутается с "канцтоварами").

Если ни одно условие не выполнено – считаем, что товара в БД нет (match_type = "none").

Это защищает от абсурдных совпадений, которые выдаёт чистый fuzzy-поиск на 600к записей.

❗ Частые проблемы и их решение
1. Вылетает после 8-10 объектов
У вас кончается память. В коде уже добавлен gc.collect() и ограничение применения ESRGAN.

Попробуйте отключить APPLY_AI_ENHANCE = False и уменьшить APPLY_CONTRAST или APPLY_DENOISE.

Если совсем плохо – уберите улучшения совсем, оставьте только enhance_roi с билатеральным фильтром.

2. OCR возвращает пустой текст
Убедитесь, что в настройках правильные YANDEX_API_KEY и YANDEX_FOLDER_ID.

Проверьте интернет‑соединение.

Иногда ценник слишком маленький – увеличьте DETECTION_SCALE = 1.5 (тогда детекция будет на увеличенном кадре, но медленнее).

3. Цена распознаётся как огромное число (674685 вместо 67₽)
Это из-за того, что OCR склеивает цифры. В parse_price_tag_text уже стоит ограничение <= 100000. Если всё равно попадается – ужесточите лимит, например до <= 10000.

4. Программа не находит видео
Проверьте путь: русские буквы и пробелы иногда мешают. Переименуйте папки и файл в латиницу.

Установите pip install opencv-python-headless, но лучше обычный opencv-python.

5. Не удаётся установить realesrgan или basicsr
Попробуйте сначала установить torch и torchvision, затем:

bash
pip install basicsr --no-deps
pip install realesrgan --no-deps
А недостающие зависимости (numpy, opencv, Pillow) уже будут.

🤝 Что делать, если что-то пошло не так?
Проверьте, что все пути в коде (к БД, модели YOLO, выходным папкам) существуют или создаются автоматически.

Загляните в консоль – программа выводит подробные логи, включая ошибки.

Отключите по очереди APPLY_AI_ENHANCE, APPLY_DENOISE, APPLY_CONTRAST, чтобы найти узкое место.

Если YOLO не детектит – проверьте файл best.pt, возможно, он повреждён или не подходит к вашим ценникам.
